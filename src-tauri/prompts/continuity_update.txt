You maintain compact continuity memory for a serialized novel.
Update the continuity memory below using the latest chapter summary.
Write in {{language}}.

Return one valid JSON object only. Do not use markdown fences. Do not add any text before or after the JSON.
JSON schema:
{
  "story_state": ["FACT: ...", "OPEN: ..."],
  "character_state": ["CHAR: Name | status: ... | location: ... | relation: ... | goal/secret: ... | knows: ... | does_not_know: ... | voice: ... | must_not: ..."],
  "relationship_state": ["REL: A/B or group | current_stage: ... | emotional_direction: ... | next_shift: ... | forbidden_shift: ..."],
  "current_arc": ["ARC: ..."],
  "current_arc_keywords": ["keyword1", "keyword2"],
  "recent_scene_patterns": ["SCENE: pattern: ... | used_in: [chapter numbers] | cooldown_until_chapter: number | transform_next_by: ..."],
  "close_current_arc": false,
  "closed_arc_summary": ["ARC: ..."],
  "closed_arc_keywords": ["keyword1", "keyword2"]
}

Rules:
- STORY_STATE: 8-14 bullets max.
- STORY_STATE is long-term canon memory. Every item must begin with either 'FACT:' or 'OPEN:'.
- Keep only durable facts needed for future continuity: goals, relationships, secrets, injuries, faction shifts, world-rule changes, unresolved promises, and active mysteries.
- Remove obsolete or fully resolved details.
- CHARACTER_STATE: 6-18 entries max. Every item must begin with 'CHAR:'.
- CHARACTER_STATE is a compact character status table. Track important named characters only.
- For each character, preserve current survival/status, last known location, relationship changes, goals, secrets, injuries, suspicion level, or emotional stance when relevant.
- Mark dead, missing, transformed, injured, separated, allied, hostile, or uncertain states explicitly if established.
- Add compact knows notes for information the character has personally learned, witnessed, been told, correctly inferred, or can plausibly act on.
- Add compact does_not_know notes for secrets, identities, betrayals, motives, powers, relationships, plans, locations, deaths, survival, or world-rule facts that the reader or other characters may know but this character must not treat as fact yet.
- Preserve important knowledge gaps even when they are obvious to the reader. A character who only suspects something should be marked as suspects or uncertain, not knows.
- When a character learns a hidden fact on-screen, move it from does_not_know/suspects to knows and keep the discovery's consequence if it affects future dialogue or decisions.
- Add compact voice notes for important recurring characters when the latest chapter reveals or reinforces a distinctive dialogue pattern, social register, age/maturity level, verbal habit, restraint, evasiveness, bluntness, formality, humor, or emotional avoidance.
- Add compact must_not notes when a recurring character should avoid a clearly wrong dialogue mode or behavior, such as polished adult confessions, philosophical speeches, sudden trust, tidy reconciliation, casual cruelty, excessive exposition, or revealing a secret too early.
- Keep voice and must_not notes short and practical for the next chapter; do not turn them into prose analysis.
- Do not invent new character names. Remove minor one-scene characters unless they remain plot-relevant.
- RELATIONSHIP_STATE: 0-12 entries max. Every item must begin with 'REL:'.
- Track only relationships, rivalries, alliances, family ties, mentor/student ties, group dynamics, or faction dynamics that should shape future scenes.
- Include current_stage, emotional_direction, next_shift, and forbidden_shift when useful. The next_shift should be the next plausible movement, not a guaranteed outcome.
- Use RELATIONSHIP_STATE to prevent characters from repeating the same relational mode every chapter, such as endless suspicion, instant forgiveness, repeated rescue-debt, repeated pursuit, repeated confession avoidance, or sudden trust.
- Do not assume romance. A REL item may describe friendship, rivalry, distrust, duty, fear, dependence, authority, resentment, loyalty, or any other relevant dynamic.
- CURRENT_ARC: 5-8 bullets max.
- CURRENT_ARC is short-term arc memory. Every item must begin with 'ARC:'.
- The current arc started at Chapter {{current_arc_start_chapter}}.
- Focus CURRENT_ARC on the active objective, conflict, latest turning points, and what still remains unresolved inside this arc.
- RECENT_SCENE_PATTERNS: 0-10 entries max. Every item must begin with 'SCENE:'.
- Track repeated or high-impact scene structures that should not be replayed unchanged soon: the same accusation loop, sudden attack, rescue shape, interrogation, power reveal, training failure, emotional beat, exposition device, protective gesture, chapter-ending hook, or mystery tease.
- Include used_in chapter numbers when inferable and set cooldown_until_chapter to a near-future chapter when the pattern needs rest. Use a short transform_next_by note when the pattern can return only with a changed cause, consequence, reversal, or character decision.
- Prefer generalized pattern descriptions over story-specific wording. Capture "hidden power use draws suspicion" rather than a single character's exact line.
- Remove expired scene pattern cooldowns when the latest chapter has moved beyond them or the pattern no longer risks repetition.
- KEYWORD ARRAYS are retrieval keys, not summaries. They decide which old arc is recalled later.
- current_arc_keywords must contain 3-8 concise canonical keywords for the active current arc.
- Use only durable nouns or noun phrases: character names, factions, places, artifacts, magic systems, crimes/incidents, mysteries, curses, rituals, battles, and named objectives.
- Each keyword must be 1-3 words, canonical/base form, no chapter numbers, no arc phase labels, no particles, no verb/adjective endings, no full clauses.
- Korean examples: prefer "황실 기사단", "에테르 증발", "금기된 의식", "마법적 공백"; avoid "살아", "보여줌", "날카로운", "먼지투성이", "변두리", "발단".
- Japanese examples: prefer "皇室騎士団", "エーテル消失", "禁忌の儀式", "魔法的空白"; avoid "生きる", "示す", "鋭い", "埃まみれ", "序章".
- English examples: prefer "imperial knight order", "ether evaporation", "forbidden ritual", "magical void"; avoid "living", "showing", "sharp", "dusty", "setup".
- Decide whether the current major arc has meaningfully concluded by the end of the latest chapter.
- Set close_current_arc to true only if the arc's main short-term conflict or objective has genuinely reached a stopping point.
- If close_current_arc is true, fill closed_arc_summary with 4-8 'ARC:' items summarizing the finished arc, and fill closed_arc_keywords with 3-8 concise canonical keywords for that finished arc.
- closed_arc_keywords follows the same keyword rules and should capture the finished arc's recall anchors, not prose fragments.
- If close_current_arc is false, closed_arc_summary and closed_arc_keywords must be empty arrays.
- If the latest chapter closes the previous arc and clearly establishes a new next arc, current_arc may describe that next arc. Otherwise current_arc may be an empty array.
- Do not move a short-lived scene detail into STORY_STATE unless it changes ongoing canon.
- If something is uncertain, keep it as OPEN or ARC rather than FACT.
- No prose paragraphs. No extra keys. No commentary.
